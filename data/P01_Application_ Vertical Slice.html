Progress Report

So far, we have identified the two sub-problems within our application:

  • Detecting face information from images of a Rubik's Cube
  • Solving the Rubik's Cube and exporting information

Since we don't care quite as much about the solving aspect for the purpose of this class, we've spent a lot more time on the color detection portion.

We've been combing through several OpenCV modules, and we've discovered that there are actually multiple ways to derive the information we're looking for from a given set of images. With that in mind, our group is currently evaluating a couple of different approaches:

  1. First, we're looking at a canny edge detection approach. Josh and Patrick have been working on implementing this in C++. Ultimately, this approach should be more dynamic and flexible, especially with respect to our end goal of taking in input through video.
Here's the input image.
Canny edge detection results.

  1. We're also evaluating a filter-based approach. Jake has been implementing this in Python. This approach is extremely simple since all it does is filter for certain color values, run the output through a threshold to minimize noise, and then do location-relative matching to determine which square on the face has a certain color.
